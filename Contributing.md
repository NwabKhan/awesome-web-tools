# Contributing Guidelines

Thanks for taking the time to contribute. Please follow these guidelines to help keep this list clean, consistent, and useful.

## Before you submit

- Search the existing list first to make sure the tool isn't already included.
- The tool must be free to use. Free-with-optional-paid-upgrade is fine; free trial or paywall-after-signup is not.
- The tool must not require account creation or login just to use its core function.
- The tool should genuinely work and be actively maintained (not abandoned or broken).

## How to add a tool

1. Fork this repository.
2. Add your entry to the relevant category, in alphabetical order within that category.
3. Use this exact format:
   ```
   - [Tool Name](https://tool-url.com) - Short, one-line, factual description.
   ```
4. Keep the description under ~100 characters, plain and factual — no marketing language, no emojis, no exclamation marks.
5. If your tool doesn't fit an existing category, open an issue first to discuss adding a new one before submitting a pull request.
6. Submit one pull request per tool. Do not bundle multiple additions into a single pull request.

## Pull request checklist

- [ ] Entry is placed in the correct category
- [ ] Entry is in alphabetical order within that category
- [ ] Description is a single line, factual, no self-promotion language
- [ ] Link goes directly to the tool, not to a homepage or landing page requiring further navigation
- [ ] Tool is free to use without signup

## What gets rejected

- Broken links or non-functional tools
- Tools requiring payment, signup, or login to use core functionality
- Duplicate entries
- Self-promotional or marketing-style descriptions
- Unrelated content (articles, courses, general resources not classified as a "tool")

## Code of conduct

Be respectful and constructive in issues and pull requests. Disagreements about categorization or inclusion are fine — keep the discussion focused on the merits of the tool itself.
